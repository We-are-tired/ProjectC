使用素材（想定）：

「SF Symbol」より、主な画像を利用。
	cloud.rain.fill - （雨雲：真っ直ぐ落ちる）
	flame.fill - （加熱：ゆっくりとジグザグに落ちる）
	battery.0 - .25 - .50 - .75 - battery.100.bolt
	（バッテリー：25%ずつチャージされ、「battery.100.bolt」で高速で真っ直ぐ落ちていく）
	trash - （ゴミ箱：初期配置画像）
	trash.fill - （ゴミ箱：上記画像の巨大化後の画像、「trash」の状態から徐々に大きくなっていく）
	iphone.homebutton - （端末(小)：自機、設定によって「iPhone」と変更可能。小文字なので注意）
	iphone - （端末(大)：自機、設定によって「iPhone.homebutton」と変更可能。小文字なので注意）
	heart - （残機：有りの状態）
	heart.fill - （残機：無しの状態）

12/07
渡辺の改修宣言に伴い、TwoUserSettingの改良。
ひとまず「GameSettingViewModel」に新しく複数の変数を宣言して、いじれるようにしている。
変数名はViewModel担当の人ないしModel担当の人に一任します。変更しても大丈夫です。
あと「速度」の部分で「１以下」「５以上」が出ないようにif文を用いてます。
コード的にイケてないって場合は、手数かけるけど改修お願いします。

またシミュレーターのプロセス数が不足してエラーが発生する場合がある模様。
ターミナルで（自分はcdでいじってるファイルまで移動してた）
sudo launchctl limit maxproc 2000 2500
上記を入力後にXcodeの再起動をすることで問題を解消しました。
